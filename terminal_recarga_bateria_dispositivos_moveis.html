INTRODUÇÃO

        Seja para comunicação ou entretenimento, estejam onde estiverem, o fato é que as pessoas não abrem mão da companhia de seus smartphones ou tablets. Entre redes sociais, jogos e outros aplicativos, a grande preocupação é a duração da bateria.

É visando uma solução eficiente para esse problema que propomos a criação de um terminal publico de recarga para dispositivos móveis, que será instalado em pontos estratégicos por onde transita uma grande quantidade de pessoas.

TERMINAL

        O objetivo do terminal é que os dispositivos sejam deixados em segurança enquanto carregam.  Para isso o  terminal terá gavetas, trancadas por uma chave que ficará com o usuário até o fim da carga. As gavetas serão sinalizadas da seguinte maneira:

        Verde       → gaveta disponível.

        Apagado  → gaveta ocupada.

        Vermelho → um dispositivo conectado, mas a gaveta está aberta.

        Amarelo   → a gaveta foi trancada sem nenhum dispositivo.

COMPUTADOR CENTRAL

Cada terminal contará com um computador central que é responsável por monitorar todo o uso do terminal. Escolhemos a Beaglebone Black, pelo baixo custo , hardware livre e que conta com processador de 1GHz, 512MB de RAM, Ethernet, microSD, microHDMI,

Iremos utilizar sistema operacional Arch Linux, pelo fato de ser focado na elegância, minimalismo e simplicidade e também por possuir um repositório de programas atualizados para arquitetura ARM.  

INSTALANDO SISTEMA OPERACIONAL

        Para instalar o Arch Linux na beaglebone Black iremos utilizar um cartão de memória microSD, que deverá ser conectado a um computador que utilize alguma distribuição Linux para fazer a formatação e instalação do sistema operacional.

Siga os passos abaixo substituindo o sdX pelo do caminho do cartão microSD.

  1. dd if=/dev/zero of=/dev/sdX bs=1M count=8
  2. fdisk /dev/sdX
  3. mkfs.ext4 /dev/sdX1
  4. mkdir mnt
  5. mount /dev/sdX1 mnt
  6. wget http://os.archlinuxarm.org/os/ArchLinuxARM-am33x-latest.tar.gz
  7. bsdtar -xpf ArchLinuxARM-am33x-latest.tar.gz -C mnt
  8. sync
  9. dd if=mnt/boot/MLO of=/dev/sdX count=1 seek=1 conv=notrunc bs=128k
  10. dd if=mnt/boot/u-boot.img of=/dev/sdX count=2 seek=1 conv=notrunc bs=384k
  11. umount mnt
  12. sync

Seguindo todos os passos basta inserir o cartão microSD na Beaglebone e segurar o botão de boot enquanto a placa é iniciada. Após inicialização do sistema utilize um terminal para se conectar via ssh alarm@192.168.7.2 .

GAVETAS

        Cada gaveta do terminal contará com conectores de carregadores mais populares  para smartphones e tablets. Além disso contarão com uma trava eletrônica que enviará um sinal para o computador do terminal informando seu status atual.

        Para que as gavetas tenham cores de acordo com seu estado serão utilizados leds RGB, que mudarão de cor ou apagarão de acordo com o estado informado pelo computador do terminal.

Para os terminais 1, 2 e 3 da imagem acima, referente as cores azul, verde e vermelho, teremos o circuito abaixo para controlar cada uma delas. Neste circuito temos uma fita de leds de uma mesma cor, o chaveamento dos leds é feito pelo transistor Q1 que é acionado pelo sinal C1 proveniente da computador central.

A detecção do dispositivo na gaveta será feita pelo sensor ACS712, que detecta a corrente AC ou DC que passa por um fio e gera um sinal de saída proporcional a corrente detectada.

A Beaglebone Black tem conversor analógico digital ADC de 12bits. A voltagem de entrada nos pinos ADCs não deve passar de 0-1.8V. O sensor de corrente ACS712 trabalha com sinal de saída entre 0-5V, para corrigir a voltagem, utilizaremos um divisor de tensão, vale salientar que o resistor conectado ao terra deverá ser menor ou igual a 1K ohm e ambos os resistores devem possuir de preferencia uma margem de erro de 0.1%.

        Analisando uma tabela de resistores comerciais encontramos valores de 1K para Z1 e 510ohm para Z2 teremos uma saída ~1.7V.

SOFTWARE

        O sistema conta com um servidor único responsável por coletar todos os dados de uso dos terminais instalados em determinados locais. A comunicação será feita por socket.

O servidor segue passos para aceitar uma conexão do cliente, primeiramente criamos socket, atribuímos um endereço utilizando o comando bind e posteriormente começamos a ouvir os pedidos de requisição dos clientes utilizando o comando listen. Para cada cliente conectado e aceito no servidor uma nova thread é gerada, a fim de permanecer com a conexão ativa. Essa conexão só é desfeita caso o cliente o desconecte. O servidor irá gravar em arquivo ou em banco de dados todos os logs recebidos dos terminais para futuras analises dos dados coletados.

        No software do cliente, há uma thread para cada gaveta, assim teremos um controle único de cada espaço do terminal e uma thread especifica para manter a conexão com o servidor ativa.

        Abaixo segue ilustrado todos os passos para estabelecer uma comunicação cliente/servidor.

        

Função

Descrição.

SOCKET

Cria novo ponto de comunicação.

BIND

Define o endereço para o socket.

LISTEN

Disponibilidade para aceitar conexões.

ACCEPT

Bloquear o servidor para estabelecer a conexão.
